# Contributing to PyNative Mobile

Thank you for your interest in contributing!  This project follows a simple
workflow:

1. Fork the repository and create a topic branch for your work.
2. Make sure your code adheres to PEP8 (`flake8`) and includes tests for new
   functionality.
3. Update documentation (`README.md`, `docs/` or other relevant files).
4. Commit your changes with a clear message and push to your fork.
5. Open a pull request against the `master` branch.

Issues and PRs are welcome for bug reports, feature ideas, documentation
improvements, and anything that makes the framework easier to use.

For major changes, please open an issue first to discuss the approach.

---

### Code Style

* Use `black` for formatting and `flake8` for linting.
* Add or update unit tests in `tests/`.
* Keep backwards compatibility within the same minor version.
